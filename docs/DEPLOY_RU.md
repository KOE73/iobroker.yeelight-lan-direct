# Инструкция по установке и обновлению на сервере ioBroker

Так как ваш сервер ioBroker находится на другой машине, следуйте этой инструкции для переноса кода.

## 1. Подготовка файлов (на вашем ПК)

Для работы адаптера на сервере необходимы следующие папки и файлы:
- `admin/` (обязательно — настройки в ioBroker)
- `core/` (обязательно — логика)
- `main.js` (обязательно)
- `io-package.json` (обязательно)
- `package.json` (обязательно)
- `panel/` (опционально — если хотите использовать отдельный дашборд)

**Лучший способ подготовки архива:**
В папке адаптера выполните команду `npm pack`. Она создаст файл вроде `iobroker.yeelight-lan-direct-0.1.0.tgz`. Именно этот файл нужно загружать на веб-сервер.

**НЕ копируйте** папки `node_modules/`, `lib/` и `.git`.

---

## 2. Первая установка на сервере

1. Перенесите файлы в любую временную папку на сервере (например, `/home/user/yeelight`).
2. Зайдите в корень ioBroker (обычно `/opt/iobroker`).
3. Выполните установку:

```bash
cd /opt/iobroker

# Установка зависимостей и регистрация пакета
npm install /home/user/yeelight

# Загрузка ресурсов админки в систему
iobroker upload yeelight-lan-direct

# Создание первого экземпляра (instance)
iobroker add yeelight-lan-direct
```

---

## 3. Обновление кода после правок

Если вы изменили код (например, логику в `core/` или верстку в `admin/`) и хотите обновить его на сервере:

### Вариант А: Быстрое обновление (только JS/HTML файлы)
Если зависимости (`package.json`) не менялись:

1. Скопируйте измененные файлы поверх старых в `/opt/iobroker/node_modules/iobroker.yeelight-lan-direct/`.
2. Если менялись файлы в папке `admin/`, обязательно выполните:
   ```bash
   iobroker upload yeelight-lan-direct
   ```
3. Перезапустите инстанс:
   ```bash
   iobroker restart yeelight-lan-direct
   ```

### Вариант Б: Полное обновление (через npm)
Если вы добавили новые библиотеки в `package.json` или поменяли структуру:

1. Замените файлы во временной папке (`/home/user/yeelight`) на новые.
2. В корне ioBroker выполните:
   ```bash
   cd /opt/iobroker
   npm install /home/user/yeelight
   iobroker upload yeelight-lan-direct
   iobroker restart yeelight-lan-direct
   ```

### Вариант В: Установка с вашего веб-сервера (URL)

Этот способ гарантирует правильную структуру архива и отсутствие ошибок при установке.

#### Способ 1: Через npm (рекомендуется)
1. В папке адаптера на вашем ПК выполните:
   ```bash
   npm pack
   ```
2. Появится файл `iobroker.yeelight-lan-direct-0.1.0.tgz`. Выложите его на веб-сервер.

#### Способ 2: Вручную через tar (если нет npm)
1. Выполните команду:
   ```powershell
   tar -czvf Yeelight.tar.gz admin core main.js io-package.json package.json
   ```
2. Файл `Yeelight.tar.gz` выложите на веб-сервер.

#### Установка в ioBroker:
1. В админке ioBroker перейдите в "Адаптеры" -> иконка "GitHub".
2. Выберите вкладку **"Custom"** (Произвольный URL).
3. Вставьте прямую ссылку на один из созданных архивов:
   - `http://192.168.199.250:6061/iob/iobroker.yeelight-lan-direct-0.1.0.tgz`
   - или `http://192.168.199.250:6061/iob/Yeelight.tar.gz`
4. Нажмите "Установить".

*Примечание: При обновлении тем же файлом ioBroker может использовать кэш. Чтобы принудительно обновить код, используйте ключ `--force` в консоли или меняйте версию в package.json.*

---

## 4. Использование GitHub (Рекомендуется)

Самый удобный способ для удаленного сервера:

1. Запуште свой проект в GitHub.
2. В админке ioBroker:
   - Перейдите в "Адаптеры".
   - Нажмите на иконку "GitHub" (установка из произвольного URL).
   - Вставьте ссылку на ваш репозиторий.
3. Для обновления просто нажимайте ту же кнопку — ioBroker сам скачает свежий код и сделает `upload`.

---

## 5. Запуск панели управления (Panel)

Чтобы панель работала на сервере:
1. Вы можете использовать `panel/panel-server.js`. 
2. Запустите его через SSH: `node panel-server.js`.
3. Теперь панель будет доступна по IP адресу вашего сервера на порту 8088 (или том, что указан в конфиге).
